1. Logging In as Admin
  1. Go to saturday-tennis.onrender.com.
  2. Enter your 4-digit player code in the login box.
  3. Click the small "admin" link below the Enter button — the password field will appear.
  4. Enter your admin password and click Enter.
  5. The red ADMIN SIDEBAR panel will appear in the left column.

2. Maintenance Mode

Temporarily prevents anyone except admins from signing up or creating teams. Use when making changes mid-week that could affect the roster.

  1. In the Admin Sidebar, click 🔒 Maintenance: OFF to turn it on.
  2. A banner will appear on the site for all users.
  3. Click again to turn it off when done.
3. Switching Play Mode

Controls how the courts and roster are organized.

  • Open Mode: All members in one combined list, sorted by signup time. Up to 6 courts of 4.
  • Split Mode: Players divided by skill level (3.0/3.5 on courts 1–3, 4.0/4.5 on courts 4–6).
  • Team Mode: Captains sign up a full court (4–8 players). Pending team requests appear for admin review.
  1. In the Admin Sidebar under Play Mode, click the mode you want to activate.
  2. The active mode shows a checkmark. Switching resets court assignments to auto.
4. Updating Session Date & Time

Sets the date and time shown at the top of the site and used by the Monday/Friday cron jobs.

  1. In the Admin Sidebar under 📅 Session Update, edit the date field (format: June 13, 2026) and time field.
  2. Click Update Info.
⚠️ Gap Week Safety: If Target Date is more than 7 days away and players are on the roster, a red warning appears at the top of the Admin Sidebar. This means the Monday cron will auto-skip — verify the date is correct before Monday 8:15 AM.
5. Resetting the Roster

Archives all current signups and clears the roster. The Monday 8:15 AM cron does this automatically each week — only use manually if needed.

  1. In the Admin Sidebar, click Reset System.
  2. Confirm the prompt. Current signups are archived with today's date.
  3. The signup-open email is not sent when done manually (only the Monday cron sends it).
6. Removing a Player from the Roster

Use when a player cannot remove themselves (e.g., they're marked NEEDS SUB or PENDING SUB and something went wrong, or they're unreachable).

  1. Find the player in the roster list on the main page.
  2. Click the small red ✕ button on the right side of their row.
  3. Confirm the dialog. The player is immediately removed from the Signups table.
  4. No email is sent to the removed player — contact them separately if needed.
If a waitlisted player moves up as a result, they are not automatically notified. You may want to send a Message Blast to the roster.
7. Court Assignments

By default, courts are assigned sequentially (players 1–4 → Court 1, etc.). You can override any court's physical number.

  1. In the Admin Sidebar, find the court assignment dropdowns next to each court label in the roster.
  2. Select a physical court number from the dropdown — it saves immediately.

To reset all assignments to auto:

  1. Click 🔄 Reset Court Assignments in the Admin Sidebar and confirm.
8. Message Blast

Sends a message to either the current roster or the entire member list.

  1. In the Admin Sidebar under 📣 Message Blast, type your message.
  2. Choose Roster Only or Entire List from the dropdown.
  3. Click Send Blast. Use sparingly — members receive this as an email.
9. Approving Applicants & Guests

New members and guests submit requests that require admin approval before they can access the site.

  1. When there are pending requests, a yellow panel appears on the main page (visible to admins only).
  2. Review the name and email, then click Approve or Deny.
  3. Approved members are added to the Master List and emailed their access code.
10. Restoring from Archive

If a roster is accidentally wiped, you can restore it from the Archive table.

  1. Make sure Target Date in Settings is set to the correct session date.
  2. Navigate directly to /restore_archive (type it in the browser URL bar).
  3. The system will deduplicate by player code and restore the most recent record per player.

To wipe the current roster without archiving first:

  1. Navigate to /wipe_signups. Use before restore if needed.
11. Gap / Bye Weeks (No Tennis)

The system auto-detects gap weeks — no checkbox or manual step is needed. Here's how it works:

  • If Target Date is more than 7 days from Monday, the Monday cron automatically skips the purge and sends a "see you on [date]" notice instead.
  • The Friday cron similarly sends a bye-week notice instead of the lock-in email.

The one thing to verify before a gap week:

  1. In Airtable Settings (or via 📅 Session Update), confirm Target Date is set to the correct next session date (not the off Saturday).
  2. That's it — the crons handle everything else.
12. Venmo Card Toggle
  1. In the Admin Sidebar, click 💙 Venmo Card: OFF/ON to show or hide the Venmo chip-in card on the main page.
